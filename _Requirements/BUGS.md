
## Блок БАГИ (Отклонения от требований)]

### Внеси в код следующие намеренные ошибки:

* Форма "Новый клиент": в поле "Адрес" вводимые символы отображаются как "___________________" (нижнее подчеркивание).

* Форма "Новый клиент": не работает кнопка "Отмена". 
    Надо, чтобы это (сама ошибка) отображалось во вкладке Console в DevTools.

* Кнопка "Отмена" в Форме редактирования клиента: Текст "Отмена" того же цвета, что и фон, из-за чего текст не видно и визуально создаётся впечатление, что на кнопке вообще нет текста. 
    То есть текст ПРИСУТСТВУЕТ, но его НЕ ВИДНО.
    
* Кнопка "Включить/выключить тёмную тему" находится в нижнем левом углу.

* При разрешении экрана 1600x900 в браузерах Microsoft Edge, Google Chrome, Mozilla Firefox (актуальные версии) кнопка «Отмена» выходит за пределы экрана (видно "мена").

* При нажатии на кнопку/ссылку "Политика конфиденциальности" (которая в футере) сама ссылка срабатывает правильно, но текст отображается в нечитабельном виде (как будто слетела кодировка).

* Заголовок формы "Новый клиент" - "Создаём клиента". 

* Футер: Опечатка в "Контактах" - Тел.: 84952223???

* Форма редактирования клиента: Можно сохранить изменения при некорректном (неполном) номере телефона в обязательном поле "Телефон".

* (API) При создании клиента с уже существующим email возвращается статус-код 201 Created. Ожидается 409 Conflict.

* (API) DELETE /delete_client возвращает 200 OK даже для несуществующего id. Ожидается 404 Not Found.

* (API) PUT /update_profile при попытке сменить email на уже существующий у другого пользователя возвращает 200 OK. Ожидается 409 Conflict.

* (API) PATCH /update_profile возвращает 403 Forbidden, если при отправке сообщения в поле "Телефон" содержится номер с недостаточным количеством символов. Ожидается 400 Bad Request.

* (API) PUT /update_client с несуществующим id возвращает 200 OK и создаёт нового клиента с этим id. Ожидается 404 Not Found.

* (API) При регистрации пользователя в ответе (в register_date) отображется неправильная дата. Число (день) и месяц - корректные, а вот год - 1970. 

### ВАЖНО:

* Swagger документация должна генерироваться по ТРЕБОВАНИЯМ.
* Реальный код должен работать с учётом блока БАГИ (т.е. содержать преднамеренные ошибки).
